# RoboQC — концепция продукта

## Проблема

На этапе **NPI (New Product Introduction)** — запуска нового изделия в
производство — качество ещё нестабильно: процессы не отлажены, дефекты
разнообразны и редки. Именно здесь возникает ловушка:

- **Ручной контроль** медленный, субъективный и не масштабируется.
- **Автоматический контроль** (АОИ, CV-модели) требует тысяч размеченных
  примеров дефектов — а на NPI их физически ещё нет.

Производитель застревает: автоматизировать нечем, а ручная проверка не успевает
за выходом на объём.

## Решение

RoboQC превращает **ручной контроль на NPI** в источник данных для
**автоматического контроля на масштабе**.

1. **Ручной сканер ошибок.** Оператор инспектирует деталь портативным сканером
   с edge-ИИ. Каждая проверка фиксируется как размеченный пример: фото, зона,
   тип дефекта — а не просто вердикт «годен / брак».
2. **Накопление контента.** Сбор обучающих данных происходит сам, внутри
   обычной работы — это не отдельный дорогой проект по разметке.
3. **Синтетическое дополнение.** Редкие дефекты, которых мало в реальности,
   досинтезируются — закрывая нехватку данных, типичную для NPI.
4. **Автоконтроль на масштабе.** К выходу на серию датасет готов; инлайн-камеры
   RoboQC инспектируют поток автоматически, обученные на ваших же деталях.

**Слоган:** «Начните вручную. Масштабируйтесь автоматически.»

## Ценность для клиента

| Боль | Что даёт RoboQC |
|---|---|
| Автоматизация не стартует — нет данных | Данные копятся с первого дня NPI |
| Сбор датасета — отдельный дорогой проект | Сбор встроен в рабочий процесс |
| Редкие дефекты не покрыты моделью | Синтетика добивает «хвост» дефектов |
| Контроль не успевает за ростом объёма | Плавный переход ручное → авто |
| Брак уходит к клиенту на ранней серии | Контроль есть уже на NPI |

## Целевые клиенты

- Контрактные производители электроники (EMS, PCB-сборка).
- Производства с частым запуском новых SKU (короткий жизненный цикл изделия).
- Инженерные/QA-команды, отвечающие за ramp-up от пилота до серии.
- На горизонте — металлообработка, литьё, сборочные операции.

## Форма продукта

- **Железо:** портативный ручной сканер ошибок (edge-ИИ на борту).
- **Софт:** облачная платформа — дашборд, библиотека дефектов, обучение
  моделей, аналитика NPI → масштаб.
- **Инлайн:** стационарные камеры RoboQC для автоматического контроля на серии.

## Технологическая основа — полный спектр CV от NVIDIA

RoboQC строится на сквозном стеке компьютерного зрения NVIDIA — см.
[architecture.md](architecture.md):

- **Jetson** — edge-инференс на ручном сканере.
- **Omniverse Replicator** — генерация синтетических дефектов.
- **TAO Toolkit** — обучение моделей детекции дефектов (transfer learning).
- **VLM / NIM-микросервисы** — описание и классификация дефектов.
- **Metropolis + DeepStream** — инлайн-инспекция потока на линии.
- **TensorRT** — инференс в темпе конвейера.

## Брендинг

Временное имя проекта — **NVLabs**, RoboQC развивается как часть
**RomeoFlexVision**. Постоянный бренд/домен определяется позже (ориентир —
сентябрь либо появление инвестора).

> Имя «NVLabs» близко к исследовательскому подразделению NVIDIA. Для внутреннего
> и питч-использования это приемлемо, но перед публичным запуском имя необходимо
> проверить на конфликт товарных знаков — особенно с учётом того, что продукт
> построен на стеке NVIDIA.
