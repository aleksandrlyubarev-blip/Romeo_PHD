# Агент «Инженер-физик» (Physics Engineer, β-ЯМР / RWDM)

Доменный агент системы Romeo PHD, понимающий предметную область диссертации:
**кинетика переноса спиновой поляризации в неупорядоченной системе ядер ⁸Li–⁶Li
в кристаллах LiF, наблюдаемая методом β-ЯМР, и её численное моделирование в рамках
модели RWDM (Random Walks in Disordered Media)** — школа Ф.С. Джепарова, ИТЭФ.

## Как это устроено в репозитории

| Компонент | Файл | Назначение |
|---|---|---|
| Персона агента | `artifacts/api-server/src/lib/agent-personas.ts` | Системный промпт с доменными знаниями; подключается к LLM-воркеру по типу узла |
| Маршрутизация | `artifacts/api-server/src/lib/pipeline-executor.ts` | Узлы с типом `physics_*` получают персону инженера-физика |
| Шаблон IDE | `artifacts/romeo-phd/src/lib/buxter.ts` | Пресет `Physics Engineer RWDM Dissertation` (кнопка **Physics** в IDE) |

Любой узел пайплайна с типом, начинающимся на `physics_`, выполняется с персоной
инженера-физика. Типы, используемые в шаблоне:

- `physics_problem_intake` — нормализация исследовательского запроса в физическую постановку;
- `physics_theory` — аудит теоретической базы и формул;
- `physics_model_design` — дизайн кинетической модели (решётка, скорости, генератор);
- `physics_simulation` — план численного эксперимента (суперячейка, конфигурации, GPU);
- `physics_validation` — валидация против опубликованных кривых и табличных параметров;
- `physics_report` — отчёт в стиле отчёта научному руководителю.

## База знаний агента

### Физическая система
- LiF; подрешётка Li — ГЦК, a = 4.026 Å, минимальное расстояние Li–Li r₀ = 2.85 Å.
- ⁸Li (I = 2, T₁/₂ ≈ 0.84 с) рождается поляризованным при захвате поляризованных нейтронов
  на ⁷Li; поляризация детектируется по асимметрии β-распада (β-ЯМР).
- ⁶Li (S = 1) случайно занимает узлы подрешётки с концентрацией c.
  Исследованная сетка: c ∈ {0.033, 0.053, 0.1006} × H₀ ∈ {0.2, 0.691, 1.21} кГс.
- H₀ ∥ [111] — наиболее узкая линия ЯМР.

### Модель RWDM
- Master equation ṗ = A·p; сохранение вероятности — нулевые суммы столбцов A.
- Скорости: w_ij ∝ ν_i (1 − 3cos²θ_ij)²/r_ij⁶ · g(Δω_ij); асимметрия каналов
  ξ = I(I+1)/[S(S+1)] = 3 для пары ⁸Li–⁶Li; каналы ⁸Li ↔ ⁶Li и ⁶Li ↔ ⁶Li.
- Наблюдаемая: автокоррелятор P₀₀(t); каноническое представление P₀₀(t) = F(t)·G(β₀t)
  с полуфеноменологической F(t) (формула (9)/(11) ЯФ-2014) и корректирующим множителем
  G(x) (формула (12)) с табулированными параметрами девяти кейсов.
- Численный метод: суперячейка + теорема Блоха (Джепаров–Львов–Шестопал), усреднение
  по случайным конфигурациям; современная реализация — PyTorch/GPU, PBC, батчи конфигураций.
- Открытый вопрос: расхождение ~2× в максимуме G(t) между расчётом 2005 г. и экспериментом
  при c = 0.1006, H₀ = 200 Гс — цель перепроверки на современных ресурсах (N_d = 4000–10000).

## Стандарты работы агента
- Аккуратность с размерностями, нормировками и конвенциями префакторов
  (они различаются между статьями и кодами).
- Обязательные проверки численных моделей: нулевые суммы столбцов A, корректность PBC
  и minimum image, сходимость по размеру суперячейки, числу конфигураций и сетке времени.
- Чёткое разделение: теория статей / феноменология / численные артефакты;
  честная фиксация расхождений с экспериментом.
- Рабочий язык — русский, терминология дублируется английскими эквивалентами.

## Как пользоваться
1. Запустите проект (`pnpm run dev`), откройте IDE.
2. Выберите пресет **Physics** — загрузится пайплайн `Physics Engineer RWDM Dissertation`.
3. При необходимости отредактируйте YAML (например, зафиксируйте конкретный кейс c × H₀
   в промпте узла `problem_intake`) и нажмите **RUN PIPELINE**.
4. Узлы со статусами `AMBIGUOUS` / `NEEDS_CLARIFICATION` уходят в очередь HITL
   (страница Consultations) — как и у остальных агентов системы.
