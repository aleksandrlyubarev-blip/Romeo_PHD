# Агент «DoskaTV Quality Control» (контроль качества раздела «Услуги»)

Доменный агент системы Romeo PHD для **контроля качества и модерации объявлений** раздела
**«Услуги»** на [DoskaTV](https://doska.tv) — доске объявлений в Израиле (200k+ пользователей,
сотни тысяч установок приложения, русскоязычная аудитория). Раздел «Услуги» развивается как
каталог мелких бизнесов и частных специалистов (репетиторы, адвокаты, бухгалтеры, фотографы,
мастера красоты, ремонтники, сантехники, клинеры, няни, психологи и др.), где **ключевую роль
играет доверие**.

## Предложение для DoskaTV (зачем платформа Romeo здесь)

Контроль качества каталога услуг — это по сути конвейер модерации, и возможности Romeo Agent
System ложатся на него один-в-один:

| Возможность платформы | Применение в модерации DoskaTV |
|---|---|
| **HITL-очередь Consultations** (пауза пайплайна + ручное решение оператора) | Очередь живых **модераторов** для спорных объявлений — лучше отдать человеку, чем ошибочно отклонить реального специалиста |
| **Confidence-маршрутизация** (`RESOLVED ≥ 0.8` / `AMBIGUOUS` / `NEEDS_CLARIFICATION`) | **Авто-аппрув** уверенных кейсов + автоматическая отправка пограничных на ручную проверку |
| **Телеметрия** (журнал событий) | **Аудит-трейл** решений модерации: что, когда и почему было пропущено/отклонено |
| **DAG-воронка** (YAML → topological sort) | Поэтапная проверка: разбор → фото → текст → доверие/SEO → решение |
| **Доменная персона по типу узла** | Зашитые правила DoskaTV к фото и контенту вместо обобщённого LLM |

Агент **не заменяет** живых модераторов, а снимает с них рутину и оставляет в очереди только
действительно спорные случаи.

## Как это устроено в репозитории

| Компонент | Файл | Назначение |
|---|---|---|
| Персона агента | `artifacts/api-server/src/lib/agent-personas.ts` | Системный промпт `DOSKA_TV_QC_PERSONA` с правилами фото/контента DoskaTV; подключается к LLM-воркеру по типу узла |
| Маршрутизация | `artifacts/api-server/src/lib/pipeline-executor.ts` → `resolvePersonaForNodeType` | Узлы с типом `qc_*` получают персону DoskaTV QC |
| Шаблон IDE | `artifacts/romeo-phd/src/lib/buxter.ts` | Пресет `DoskaTV Quality Control` (кнопка **DoskaTV** в IDE) |

Любой узел пайплайна с типом, начинающимся на `qc_`, выполняется с персоной контроля качества.
Типы, используемые в шаблоне:

- `qc_listing_intake` — нормализация объявления (заголовок, описание, категория, география, цена, контакты, фото);
- `qc_photo_authenticity` — подлинность фото: ИИ-генерация, стоки, надписи/водяные знаки, тяжёлые фильтры, реклама;
- `qc_photo_quality` — качество и релевантность фото: реальный портрет + фото процесса/результата там, где применимо;
- `qc_content_compliance` — корректность категории, географии, формата цены, контактов; отсутствие спама;
- `qc_trust_seo` — оценка доверия для каталога + SEO под Google и пригодность к переводу на иврит (מציאה);
- `qc_moderation_decision` — решение approve / reject / ручная модерация + фидбэк специалисту.

## База знаний агента

### Правила к фотографиям (обязательно)
- Фото специалиста — **качественное реальное**: без ИИ, без надписей/водяных знаков/текстовых
  оверлеев/логотипов, без тяжёлых фильтров и чрезмерной ретуши, без рекламных баннеров и стоков.
- **Желательны** фото рабочего процесса/результата там, где применимо (фотограф, мастер красоты,
  ремонтник, строитель, массажист, клинер, мастер по технике) — они показывают «как вы работаете».
- Минимум ложных срабатываний на живых любительских фото: «нестудийное» ≠ «плохое».

### Содержание объявления
- Корректная **категория услуги** (соответствует тексту и фото).
- **География работы**: точный адрес / город / регион / вся страна / онлайн.
- **Формат цены**: «за час», «за консультацию», «за сеанс» или «договорная».
- Валидные контакты (телефон, мессенджеры); без спама, дублей и мошеннического контента.

### Доверие, SEO и иврит
- Доверие — ключевая ценность раздела: полнота профиля, реальное фото, ясность услуги.
- Органический трафик Google: заголовок/описание естественно отражают связку «услуга + город»
  (напр. «репетитор по математике в Хайфе», «русскоязычный адвокат в Бат-Яме») без переспама.
- Готовность к переводу на иврит и публикации на ивритоязычной доске **מציאה**.

## Стандарты работы агента
- Вердикт по каждой проверке = **причина + конкретная рекомендация** специалисту (по-русски,
  доброжелательно: размещение бесплатное, цель — помочь бизнесу).
- Доказательность: перечислять конкретные признаки (что указывает на ИИ/фильтр/несоответствие),
  а не общие формулировки.
- Честная калибровка уверенности: явные нарушения — высокая уверенность; пограничные кейсы —
  на **ручную модерацию** (HITL), а не авто-отклонение.

## Как пользоваться
1. Запустите проект (`pnpm run dev`), откройте IDE.
2. Выберите пресет **DoskaTV** — загрузится пайплайн `DoskaTV Quality Control`.
3. При необходимости отредактируйте YAML (например, вставьте в промпт узла `listing_intake`
   текст и описание фото конкретного объявления) и нажмите **RUN PIPELINE**.
4. Узлы со статусами `AMBIGUOUS` / `NEEDS_CLARIFICATION` уходят в очередь HITL
   (страница Consultations) — это и есть очередь ручной модерации; события видны в Telemetry.
